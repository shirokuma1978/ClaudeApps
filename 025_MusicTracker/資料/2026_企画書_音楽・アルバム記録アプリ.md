# 音楽・アルバム記録アプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

音楽愛好家は聴いたアルバムや気に入った曲を記録・整理したいと考えている。音楽・アルバム記録アプリは以下の課題を解決する。

- 聴いたアルバム・楽曲の記録と評価を蓄積する
- 好みのアーティスト・ジャンルの傾向を分析する
- まだ聴いていないアルバムのウィッシュリストを管理する

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身（個人利用） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | 音楽好き同士のレビュー共有 |

---

## 3. 主要機能

### 3.1 アルバム・楽曲記録機能（MVP）
- アルバム・楽曲の登録・編集・削除
- 登録項目：
  - タイトル（アルバム/楽曲名）
  - アーティスト
  - ジャンル
  - リリース年
  - 聴いた日
  - 評価（5段階）
  - レビュー・感想
  - タグ（ムード・シーンなど）

### 3.2 Music API連携（Phase 2）
- MusicBrainz / Last.fm API でアルバム情報自動取得
- ジャケット画像の取得

### 3.3 ウィッシュリスト
- 聴きたいアルバムの登録
- 優先度設定
- ストリーミングサービスへのリンク

### 3.4 統計・可視化
- アーティスト別・ジャンル別の聴取数
- 年別のアルバム聴取数
- 好みの傾向分析

### 3.5 プレイリスト管理（Phase 3）
- シーン別・ムード別プレイリスト
- 友人へのプレイリスト共有

---

## 4. 非機能要件

|-----|-----|

## 5. 画面構成（概要）

```
├── ログイン画面
├── ユーザー登録画面
├── ホーム（最近の記録・おすすめ再聴）
├── 記録一覧（アルバム/アーティスト別）
├── アルバム登録・編集画面
├── 詳細・レビュー画面
├── ウィッシュリスト画面
├── 統計・分析画面
└── 設定画面
```
