# アレルギー管理アプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

食物アレルギーや薬物アレルギーを持つ人が増加しており、外食・処方時のリスク管理が重要である。アレルギー管理アプリは以下の課題を解決する。

- 自分・家族のアレルゲン情報を一元管理する
- 外食時・処方時にアレルギー情報をすぐに提示できる
- アレルギー反応の記録・パターン分析で原因特定を支援する

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身・家族（個人利用） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | 医療機関への情報提供 |

---

## 3. 主要機能

### 3.1 アレルゲン登録機能（MVP）
- アレルゲンの登録・編集・削除
- 登録項目：
  - アレルゲン名（食品・薬物・環境等）
  - 種類（食物・薬物・花粉・動物等）
  - 重症度（軽症・中等症・重症・アナフィラキシー）
  - 確認方法（検査済み・経験則）
  - 代替品・注意事項メモ

### 3.2 アレルギーカード機能
- 緊急時に提示できるアレルギーカードの表示
- QRコード生成での共有
- 多言語表示（英語等）

### 3.3 反応記録機能
- アレルギー反応の記録
- 症状・原因食品・対処法の記録
- 医師への報告用情報の蓄積

### 3.4 食品チェック機能（Phase 2）
- 食品の原材料入力でのアレルゲン検出
- 注意すべき食品リストの管理

### 3.5 医療機関連携（Phase 3）
- かかりつけ医へのアレルギー情報共有
- 薬局での処方前確認用データ提供

---

## 4. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | スマホ・PCブラウザ対応 |
| 緊急表示 | アレルギーカードはオフラインでも表示 |
| データ保存 | ユーザーごとのデータ永続化 |
| セキュリティ | 医療情報の適切な保護 |
| パフォーマンス | アレルギーカード表示は即時 |

```
├── 反応記録入力画面

```
