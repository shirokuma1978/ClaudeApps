# ポイント管理アプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

現代人は楽天ポイント・dポイント・Tポイント・PayPayポイントなど多数のポイントを保有しているが、管理が煩雑で有効期限切れで失効させてしまうことが多い。ポイント管理アプリは以下の課題を解決する。

- 複数のポイントサービスの残高を一元管理する
- 有効期限が近いポイントをアラートで通知する
- 各サービスでのポイント獲得・使用を記録して最適化する

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身（個人利用） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | 家族でのポイント共有管理 |

---

## 3. 主要機能

### 3.1 ポイント登録機能（MVP）
- ポイントサービスの登録・管理
- 登録項目：
  - サービス名（楽天・d・T・PayPay等）
  - 現在の残高
  - 有効期限
  - ポイント種別（現金同等・利用制限あり）
  - メモ

### 3.2 ポイント増減記録
- ポイントの獲得・使用の記録
- 増減履歴の管理
- 残高の自動更新

### 3.3 有効期限アラート
- 有効期限30日前・7日前・1日前のアラート
- 期限切れ間近ポイントの一覧表示
- おすすめ使用方法の提示

### 3.4 統計・可視化
- 全サービス合計ポイントの円グラフ
- ポイント獲得・消費の推移
- 失効ポイントの記録

### 3.5 おすすめ活用提案（Phase 3）
- ポイントの最適活用シナリオ
- ポイント還元率の高いキャンペーン情報

---

## 4. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | スマホ・PCブラウザ対応 |
| データ保存 | ユーザーごとのデータ永続化 |
| 通知 | ブラウザ通知による期限アラート |
| パフォーマンス | ダッシュボード表示は2秒以内 |
| 汎用性 | 任意のポイントサービスに対応 |

## 5. 画面構成（概要）

```
├── ログイン画面
├── ユーザー登録画面
├── ホーム（総保有ポイント・期限切れ間近）
├── ポイントサービス一覧画面
├── ポイント残高入力画面
├── 増減記録入力画面
├── 期限アラート画面
├── 統計・グラフ画面
└── 設定画面（通知設定・サービス管理）
```
