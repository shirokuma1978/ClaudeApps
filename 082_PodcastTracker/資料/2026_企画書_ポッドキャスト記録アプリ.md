# ポッドキャスト記録アプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

ポッドキャストを愛聴するリスナーが増える中、気になるエピソードの管理・メモが課題となっている。ポッドキャスト記録アプリは以下の課題を解決する。

- 聴いたエピソードの記録・評価をストックする
- 気になるポイントのメモを取って知識を定着させる
- お気に入り番組・エピソードを管理してレコメンドに活かす

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身（ポッドキャストリスナー） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | ポッドキャスト好きとの情報共有 |

---

## 3. 主要機能

### 3.1 エピソード記録機能（MVP）
- 聴いたエピソードの登録
- 登録項目：
  - 番組名
  - エピソードタイトル
  - 配信日・再生時間
  - 聴いた日
  - 評価（5段階）
  - カテゴリ（ビジネス・教育・エンタメ等）
  - メモ・学び
  - URL・RSS

### 3.2 番組管理
- お気に入り番組のリスト
- 新着エピソードのURL管理
- 聴取ステータス（購読中・聴き終わった・中断）

### 3.3 リスニングログ
- 週間・月間の聴取時間グラフ
- カテゴリ別の内訳
- 学びのメモ一覧

### 3.4 メモ・ノート機能（Phase 2）
- エピソードのハイライト管理
- 聞きたい質問・考察のメモ
- タグによる知識の整理

### 3.5 レコメンデーション（Phase 3）
- 好みの傾向分析によるエピソード提案
- 仲間へのおすすめシェア

---

## 4. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | スマホ・PCブラウザ対応 |
| データ保存 | ユーザーごとのデータ永続化 |
| 検索 | 番組名・タグ・キーワードでの検索 |
| パフォーマンス | 一覧表示は3秒以内 |
| リンク | Spotify・Apple Podcast等へのリンク |

## 5. 画面構成（概要）

```
├── ログイン画面
├── ユーザー登録画面

├── エピソード一覧・検索
├── エピソード登録・編集画面
├── 番組管理画面
├── 統計・グラフ画面
└── 設定画面
```
