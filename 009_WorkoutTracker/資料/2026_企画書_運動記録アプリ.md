# 運動記録アプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

継続的な運動習慣を維持するには、記録を通じた可視化が効果的である。運動記録アプリは以下の課題を解決する。

- ワークアウトの内容・成果を記録し、成長を実感できる
- 種目別のパーソナルベスト（PB）を管理する
- 運動頻度・消費カロリーを把握して健康管理に役立てる

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身（個人利用） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | トレーニング仲間との記録共有・競い合い |

---

## 3. 主要機能

### 3.1 ワークアウト記録機能（MVP）
- ワークアウトの登録・編集・削除
- 登録項目：
  - 日付・時間
  - 運動種目（筋トレ・ランニング・水泳など）
  - セット数・レップ数・重量（筋トレ）
  - 距離・タイム・ペース（有酸素）
  - 消費カロリー
  - RPE（主観的運動強度）
  - メモ

### 3.2 ワークアウトテンプレート
- よく使うメニューのテンプレート保存
- 前回のトレーニングを参照・コピー

### 3.3 進捗・PB管理機能
- 種目別のパーソナルベスト表示
- 重量・距離の推移グラフ
- 前回比の表示

### 3.4 統計・可視化機能
- 週間・月間の運動頻度カレンダー
- 部位別のトレーニング量分析
- 総消費カロリーの推移

### 3.5 プログラム管理（Phase 3）
- 週次トレーニングプログラムの作成
- プログラムの進捗管理

---

## 4. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | スマホブラウザ（Web）優先 |
| レスポンシブ | ジム内でのスマホ操作を考慮したUI |
| データ保存 | ユーザーごとのデータ永続化 |
| パフォーマンス | セット間の素早い記録入力（1秒以内） |
| オフライン | ジムでのネット不安定に対応（将来） |

## 5. 画面構成（概要）

```
├── ログイン画面
├── ユーザー登録画面
├── ホーム（今日のワークアウト・最近の記録）

├── ワークアウト一覧画面
├── 種目別進捗画面
├── 統計・グラフ画面
├── テンプレート管理画面
└── 設定画面
```
