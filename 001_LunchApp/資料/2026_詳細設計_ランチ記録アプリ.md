### 1.2 共通API処理（api.js）

全リクエストのAuthorizationヘッダーに `Bearer {token}` を付与する。

| 関数名 | 引数 | 戻り値 | 概要 |
|-------|------|-------|-----|
| getRecords() | なし | Record[] | 記録一覧取得 |
| getRecord(id) | id: int | Record | 記録1件取得 |
| createRecord(data) | data: object | Record | 記録登録 |
| updateRecord(id, data) | id: int, data: object | Record | 記録更新 |
| deleteRecord(id) | id: int | void | 記録削除 |
| searchRecords(query) | query: object | Record[] | 検索 |
| getMonthlySummary() | なし | object | 当月合計取得 |

### 1.3 認証処理（auth.js）

| 関数名 | 引数 | 戻り値 | 概要 |
|-------|------|-------|-----|
| register(name, email, password) | string×3 | Token | ユーザー登録＋自動ログイン |
| login(email, password) | string×2 | Token | ログイン・トークン保存 |
| logout() | なし | void | トークン削除・ログイン画面へ |
| requireAuth() | なし | boolean | 未ログイン時にリダイレクト |
| getToken() | なし | string | localStorageからトークン取得 |

### 2.2 APIエンドポイント詳細

#### POST /auth/register
- 概要：ユーザー登録
- リクエストボディ：`{ name, email, password }`
- パスワードは6文字以上必須
- メールアドレス重複時：400を返す
- レスポンス例：
```json
{
  "access_token": "eyJ...",
  "token_type": "bearer",
  "user": { "id": 1, "name": "田中", "email": "tanaka@example.com", "created_at": "..." }
}
```

#### POST /auth/login
- 概要：ログイン・JWTトークン発行
- リクエストボディ：`{ email, password }`
- 認証失敗時：401を返す

#### GET /auth/me
- 概要：ログイン中のユーザー情報取得
- 認証：Bearerトークン必須

#### GET /api/records
- 概要：ログインユーザーの記録一覧を日付降順で取得
- 認証：Bearerトークン必須
- レスポンス例：
```json
[
  {
    "id": 1,
    "date": "2026-03-25",
    "shop_name": "松屋",
    "menu_name": "牛めし",
    "price": 580,
    "rating": 4,
    "area": "新橋"
  }
]
```

#### POST /api/records
- 概要：記録を新規登録（ログインユーザーに紐付け）
- 認証：Bearerトークン必須
- リクエストボディ：RecordCreateスキーマ
- レスポンス：登録したRecordオブジェクト
- バリデーションエラー時：422を返す

#### GET /api/records/{id}
- 概要：指定IDの記録を取得（自分の記録のみ）
- 認証：Bearerトークン必須
- 存在しない／他人の記録の場合：404を返す

#### PUT /api/records/{id}
- 概要：指定IDの記録を更新（自分の記録のみ）
- 認証：Bearerトークン必須
- 存在しない／他人の記録の場合：404を返す

#### DELETE /api/records/{id}
- 概要：指定IDの記録を削除（自分の記録のみ）
- 認証：Bearerトークン必須
- 存在しない／他人の記録の場合：404を返す

#### GET /api/records/search
- 概要：ログインユーザーの記録を検索
- 認証：Bearerトークン必須
- クエリパラメータ：
  - keyword: string（店名・メニュー名の部分一致）
  - area: string（完全一致）
- レスポンス：条件に合うRecord[]

#### GET /api/summary/monthly
- 概要：ログインユーザーの当月合計金額・件数を返す
- 認証：Bearerトークン必須
- レスポンス例：
```json
{
  "month": "2026-03",
  "total_price": 12400,
  "count": 18
}
```

### 2.3 モデル定義（models.py）

```python
class User(Base):
    __tablename__ = "users"

    id              = Column(Integer, primary_key=True, autoincrement=True)
    name            = Column(String(100), nullable=False)
    email           = Column(String(255), nullable=False, unique=True, index=True)
    hashed_password = Column(String(255), nullable=False)
    created_at      = Column(DateTime, default=datetime.utcnow, nullable=False)

    records = relationship("LunchRecord", back_populates="owner")


class LunchRecord(Base):
    __tablename__ = "lunch_records"

    id          = Column(Integer, primary_key=True, autoincrement=True)
    user_id     = Column(Integer, ForeignKey("users.id"), nullable=False, index=True)
    date        = Column(Date, nullable=False)
    time_of_day = Column(String(10))
    shop_name   = Column(String(100), nullable=False)
    menu_name   = Column(String(100))
    price       = Column(Integer)
    area        = Column(String(50))
    companions  = Column(String(100))
    rating      = Column(Integer)
    memo        = Column(Text)
    created_at  = Column(DateTime, default=datetime.utcnow, nullable=False)
    updated_at  = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow, nullable=False)

    owner = relationship("User", back_populates="records")
```

### 2.4 スキーマ定義（schemas.py）

# 記録スキーマ
class RecordBase(BaseModel):
    date: date
    time_of_day: Optional[str] = None
    shop_name: str
    menu_name: Optional[str] = None
    price: Optional[int] = None
    area: Optional[str] = None
    companions: Optional[str] = None
    rating: Optional[int] = None

class RecordCreate(RecordBase): pass
class RecordUpdate(RecordBase): pass

class Record(RecordBase):
    id: int
    created_at: datetime
    updated_at: datetime
    model_config = {"from_attributes": True}

# 認証スキーマ（Phase 2）

## 4. UI/UX設計方針

### 4.1 基本方針

**「食べた直後に30秒で記録できる」を最優先とする。**

- 必須入力は店名と日付だけ。他は全部任意
- タップ数を最小限に
- スマホ片手操作を前提にしたレイアウト

### 4.2 入力の自動化・省力化

| 項目 | 自動化・省力化の内容 |
|-----|-------------------|
| 日付 | デフォルト：今日の日付を自動セット |
| 時間帯 | 現在時刻で自動判定（10〜15時→昼、17時以降→夜、それ以外→その他） |
| 店名 | 過去に入力した店名をサジェスト表示（前方一致） |
| エリア | 直前に使ったエリアをデフォルト表示 |
| 評価 | 星タップ1回で入力完結（デフォルト：未選択） |
| 金額 | 数字キーボードを自動表示 |

### 4.3 初期値一覧

| 項目 | 初期値 | 備考 |
|-----|-------|-----|
| 日付 | 今日 | 変更可 |
| 時間帯 | 時刻で自動判定 | 変更可 |
| 店名 | 空欄 | 過去履歴サジェスト |
| メニュー名 | 空欄 | - |
| 金額 | 空欄 | 数字キーボード表示 |
| エリア | 直前に使ったエリア | 未使用時は空欄 |
| 同行者 | 空欄 | - |
| 評価 | 未選択 | - |
| メモ | 空欄 | - |

### 4.4 操作性の工夫

- **登録ボタン（FAB）**：ホーム右下に固定表示。どこからでも1タップで登録画面へ
- **保存後の遷移**：保存完了後はホームに戻り、登録した記録が先頭に表示される
- **削除確認**：削除ボタンタップ後に確認ダイアログを表示（誤削除防止）
- **キーボード**：金額欄は数字キーボード、他はテキストキーボードを自動表示
- **スクロール**：一覧は無限スクロール（ページネーションなし）

### 4.5 デザイン方針

- フォントは大きめ（16px以上）でスマホで読みやすく
- カード間の余白を十分に取り、誤タップを防ぐ
- 評価は星アイコンで視覚的にわかりやすく
- 金額・評価は一覧カードに必ず表示（一目でわかるように）

## 7. エラーハンドリング方針

| HTTPステータス | 場面 |
|-------------|-----|
| 200 | 正常 |
| 201 | 登録成功 |
| 400 | メールアドレス重複など業務エラー |
| 401 | 未認証（トークンなし・期限切れ・不正） |
| 404 | 指定IDが存在しない／他人の記録 |
| 422 | バリデーションエラー |
| 500 | サーバーエラー |
