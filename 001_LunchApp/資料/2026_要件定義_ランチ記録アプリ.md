# ランチ記録アプリ 要件定義

**バージョン:** 2.0
**作成日:** 2026年3月25日
**更新日:** 2026年3月26日
**対象フェーズ:** Phase 1（MVP）+ Phase 2（ユーザー認証）

---

## 1. ユーザーストーリー

| # | ストーリー | フェーズ |
|---|-----------|---------|
| US01 | ランチを食べたあと、店名・メニュー・金額・評価をすぐに記録したい | Phase 1 |
| US02 | 過去の記録を一覧で見たい | Phase 1 |
| US03 | 「あの店なんだっけ」と思ったとき、店名やエリアで検索したい | Phase 1 |
| US04 | 今月いくらランチに使ったか確認したい | Phase 1 |
| US05 | アカウントを作成して自分だけの記録を管理したい | Phase 2 |

| US07 | 他のユーザーの記録が見えないようにしたい | Phase 2 |

## 2. 機能要件

### 2.1 記録登録
- 以下の項目を入力して保存できる
  - 日付（必須）
  - 時間帯（昼・夜など）
  - 店名（必須）
  - メニュー名
  - 金額
  - 場所・エリア
  - 一緒に行った人
  - 評価（1〜5）
  - メモ

### 2.2 一覧表示
- 記録を日付の新しい順に表示する
- 1件あたり：日付・店名・金額・評価を表示する

### 2.3 詳細表示
- 一覧から1件タップで全項目を表示する

### 2.4 編集・削除
- 登録済みの記録を編集・削除できる

### 2.5 検索・フィルター
- 店名・メニュー名のキーワード検索
- エリアで絞り込み

### 2.6 集計
- 当月の合計金額を表示する

### 2.7 ユーザー認証（Phase 2）
- ユーザー登録
  - 名前・メールアドレス・パスワード（6文字以上）を入力して登録できる
  - メールアドレスの重複チェックを行う
- ログイン
  - メールアドレス＋パスワードで認証する
  - 認証成功時にJWTトークンを発行し、ブラウザに保存する
- ログアウト
  - トークンを削除してログイン画面へ戻る
- アクセス制御
  - 未ログイン状態でのページアクセスはログイン画面へリダイレクト
  - 各記録はログインユーザー自身のものだけを表示・操作できる

---

## 3. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | スマホブラウザ（Chrome）優先 |
| データ保存 | サーバーサイドDB（SQLite／PostgreSQL）で永続化 |
| 表示速度 | 一覧表示は2秒以内 |
| 認証トークン | JWT（有効期限30日）、localStorageに保存 |
| パスワード | bcryptでハッシュ化して保存 |

---

## 4. 対象外

### Phase 1・2 では作らない（Phase 3以降）
- 友人との共有
- 写真添付
- 地図連携
