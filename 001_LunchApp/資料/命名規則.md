# ファイル命名規則

**作成日:** 2026年3月25日

---

## 1. 基本ルール

### アイデア・検討段階
```
YYYY_工程名_プロジェクト名.md
```
例：`2026_企画書_ランチ記録アプリ.md`

### 採用・開発開始後
```
YYYYPP_NN_工程名_プロジェクト名.md
```
例：`202601_01_企画書_ランチ記録アプリ.md`

---

## 2. 各要素の意味

| 要素 | 桁数 | 内容 | 例 |
|-----|------|-----|-----|
| YYYY | 4桁 | プロジェクト開始年 | 2026 |
| PP | 2桁 | プロジェクトID（採用後に採番） | 01, 02, 03… |
| NN | 2桁 | 工程番号 | 01, 02, 03… |
| 工程名 | - | ドキュメント種別 | 企画書、要件定義… |
| プロジェクト名 | - | プロジェクト識別名 | ランチ記録アプリ |

---

## 3. 工程番号の標準

| 番号 | 工程名 |
|-----|-------|
| 01 | 企画書 |
| 02 | 要件定義 |
| 03 | 基本設計 |
| 04 | 詳細設計 |
| 05 | 製造 |
| 06 | 単体テスト |
| 07 | 結合テスト |
| 08 | 総合テスト |

---

## 4. プロジェクトID管理

| ID | プロジェクト名 | 開始日 | ステータス |
|----|-------------|-------|---------|
| 未採番 | ランチ記録アプリ | 2026/03/25 | 検討中 |

採用決定時にIDを採番し、このテーブルを更新する。

---

## 5. 採用決定時のリネーム例

```
2026_企画書_ランチ記録アプリ.md
　　↓ プロジェクトID=01 で採用決定
202601_01_企画書_ランチ記録アプリ.md
```

---

*このファイルはプロジェクト共通の命名規則として管理する。*
