# 読書記録アプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

読んだ本の記録を残すことで、知識の整理・振り返りができ、読書習慣の継続にもつながる。読書記録アプリは以下の課題を解決する。

- 読了した本・読んでいる本・積読の管理を一元化する
- 本から得た気づき・メモを記録し、後から参照できるようにする
- 年間の読書冊数・ジャンル傾向を可視化する

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身（個人利用） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | 友人との本の貸し借り・おすすめ共有 |

---

## 3. 主要機能

### 3.1 本の登録機能（MVP）
- 本の登録・編集・削除
- 登録項目：
  - タイトル・著者
  - ISBN（バーコードスキャン将来対応）
  - ジャンル・カテゴリ
  - 読書ステータス（読みたい・読中・読了・中断）
  - 評価（5段階）
  - 読了日・読書期間
  - 感想・メモ

### 3.2 本棚ビュー
- ステータス別の本棚表示
- カバー画像の表示（ISBN連携）
- グリッド表示・リスト表示の切り替え

### 3.3 読書ログ機能
- ページ数・進捗の記録
- 読書セッションのタイムログ
- 気づき・引用のメモ

### 3.4 統計・可視化機能
- 年間・月間読書冊数
- ジャンル別内訳グラフ
- 読書ペースの推移

### 3.5 おすすめ・共有機能（Phase 3）
- 本のレビュー共有
- 友人へのおすすめ機能
- 読書リストの公開設定

---

## 4. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | スマホ・PCブラウザ対応 |
| 外部API | Google Books API または OpenBD を使った書籍情報取得 |
| データ保存 | ユーザーごとのデータ永続化 |
| パフォーマンス | 書籍情報取得は5秒以内 |
| 画像 | 書影のキャッシュ・最適化 |

---

## 5. 画面構成（概要）

```
├── ログイン画面
├── ユーザー登録画面
├── ホーム（読中の本・最近の記録）
├── 本棚画面（ステータス別タブ）
├── 本の登録・編集画面
├── 本の詳細画面（メモ・進捗）
├── 統計画面
├── 検索画面（タイトル・著者・ISBN）
└── 設定画面
```
