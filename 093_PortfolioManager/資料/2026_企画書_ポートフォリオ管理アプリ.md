# ポートフォリオ管理アプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

エンジニア・デザイナー・クリエイターが自分の作品・実績をまとめたポートフォリオを管理・公開することは、キャリアアップに不可欠である。ポートフォリオ管理アプリは以下の課題を解決する。

- 自分の作品・プロジェクト実績を一元管理する
- 採用担当者・クライアントに見せるためのポートフォリオページを管理する
- 実績の継続的な更新と整理をする

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身（エンジニア・デザイナー・クリエイター） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | エージェント・採用担当者への公開共有 |

---

## 3. 主要機能

### 3.1 プロジェクト登録機能（MVP）
- プロジェクト・作品の登録・編集・削除
- 登録項目：
  - プロジェクト名
  - 概要・課題解決のポイント
  - 使用技術・ツール
  - 担当範囲・役割
  - 期間
  - 成果・実績（数値）
  - URL（ライブサイト・GitHub等）
  - スクリーンショット・画像
  - 公開/非公開設定

### 3.2 スキルタグ管理
- 技術スタック・ツールのタグ管理
- プロジェクトとスキルの紐付け
- スキル別のプロジェクト検索

### 3.3 ポートフォリオ整理
- 公開用ポートフォリオの選定
- 年代・カテゴリ別の整理
- ベスト作品のピン留め

### 3.4 公開ポートフォリオページ（Phase 2）
- 公開URLでのポートフォリオ表示
- カスタマイズ可能なテンプレート
- 自己紹介・連絡先ページ

### 3.5 採用・提案への活用（Phase 3）
- 特定案件向けのカスタマイズポートフォリオ
- 閲覧履歴・アクセス解析

---

## 4. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | PC・スマホブラウザ対応 |
| 公開ページ | 外部公開用のポートフォリオURL |
| 画像 | 作品スクリーンショットのアップロード |
| データ保存 | ユーザーごとのデータ永続化 |
| パフォーマンス | ポートフォリオページは3秒以内 |

## 5. 画面構成（概要）

```
├── ログイン画面
├── ユーザー登録画面
├── ホーム（プロジェクト一覧・公開ページプレビュー）
├── プロジェクト登録・編集画面
├── スキルタグ管理画面
├── 公開ポートフォリオ設定画面
├── 公開ページ（外部向け）
└── 設定画面
```

|---------|---------|
| フロントエンド | HTML / CSS / JavaScript |
| バックエンド | Python（FastAPI） |

| データベース | SQLite（開発）→ PostgreSQL（本番） |
