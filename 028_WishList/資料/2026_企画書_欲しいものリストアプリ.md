# 欲しいものリストアプリ 企画書

**バージョン:** 1.0
**作成日:** 2026年3月27日
**ステータス:** 企画中

---

## 1. 背景・目的

「これ欲しい！」と思った瞬間をすぐに記録し、購入判断を後から冷静に行えるようにする。欲しいものリストアプリは以下の課題を解決する。

- 衝動買いを防ぎ、本当に欲しいものだけを購入する意思決定を支援する
- 誕生日・クリスマスなどの際に家族・友人にリストを共有できる
- 購入済みアイテムの満足度を記録して将来の判断に活かす

---

## 2. ターゲットユーザー

| フェーズ | ユーザー |
|---------|---------|
| Phase 1（初期） | 自分自身（個人利用） |
| Phase 2（認証） | ユーザー登録・ログインによるマルチユーザー対応 |
| Phase 3（拡張） | 家族・友人へのリスト公開・プレゼント選び支援 |

---

## 3. 主要機能

### 3.1 アイテム登録機能（MVP）
- アイテムの登録・編集・削除
- 登録項目：
  - 商品名
  - カテゴリ（電子機器・ファッション・本・その他）
  - 価格
  - 購入予定ショップ・URL
  - 優先度（高・中・低）
  - 理由・メモ
  - 登録日
  - ステータス（欲しい・購入済み・諦めた）

### 3.2 購入待ち期間設定
- 登録から一定期間後（例：2週間）に本当に欲しいか確認通知
- 衝動買い防止のための「冷却期間」設定

### 3.3 優先度・予算管理
- 月の購入予算設定
- 優先度順のリスト表示
- 予算内で買えるアイテムのハイライト

### 3.4 購入後フィードバック
- 購入したアイテムの満足度記録
- コストパフォーマンス評価
- 後悔度の記録

### 3.5 共有機能（Phase 3）
- リストのURL共有
- 誕生日・プレゼント用リストの作成
- 友人からのプレゼント予約機能

---

## 4. 非機能要件

| 項目 | 内容 |
|-----|-----|
| 対応環境 | スマホ・PCブラウザ対応 |
| URLスクレイピング | 商品URLから名前・価格・画像を自動取得 |
| データ保存 | ユーザーごとのデータ永続化 |
| パフォーマンス | 登録は1秒以内 |
| 通知 | 冷却期間終了の通知 |

---

## 5. 画面構成（概要）

```
├── ログイン画面
├── ユーザー登録画面
├── ホーム（欲しいものリスト・今月の予算残高）
├── アイテム登録・編集画面
├── アイテム詳細画面
├── カテゴリ別一覧画面
├── 購入済みアイテム・満足度記録画面
├── 共有リスト画面
└── 設定画面（予算・冷却期間設定）
```
