# 公園宝探しアプリ（KouenQuest）基本設計

**バージョン:** 1.0
**作成日:** 2026年3月28日
**対象フェーズ:** Phase 1（MVP）

---

## 1. システム構成

```
フロントエンド（HTML / CSS / JavaScript）
　　↓ HTTP / Geolocation API / Web Audio API
バックエンド（Python FastAPI）
　　↓
データベース（SQLite ローカル / PostgreSQL 本番）
　　↑
地図（OpenStreetMap / Leaflet.js）※宝セット画面のみ
```

* フロントエンドはSPA構成（シングルページアプリケーション）
* レーダー・クエスト・シェンロンの3画面はタブで切り替え
* 欲しいものリスト・アルバムのみDBに永続化
* レーダー・クエスト・シェンロンのセッションデータはメモリ管理（localStorage補助）

---

## 2. 画面遷移図

```
flowchart TD
    HOME[ホーム画面] --> WISH[欲しいものリスト画面]
    HOME --> SETUP[宝セット画面（親）]
    WISH --> HOME
    SETUP --> RADAR[冒険画面\nタブ：レーダー／クエスト／シェンロン／設定]
    RADAR --> HOME
    RADAR --> ALBUM[アルバム画面]
    ALBUM --> HOME
```

* 未承認のお願いごとがある場合、ホームからその旨を表示
* 宝セット完了後に冒険画面へ遷移
* シェンロン召喚→アルバム記録完了後はホームに戻る

```
erDiagram
```

## 4. API設計

### 欲しいものリスト（/api/wishes）

| メソッド | パス | 概要 |
| --- | --- | --- |
| GET | /api/wishes | 一覧取得（欲しい度降順） |
| POST | /api/wishes | 新規登録 |
| PUT | /api/wishes/{id} | 内容・欲しい度・承認状態の更新 |
| DELETE | /api/wishes/{id} | 削除 |
| PUT | /api/wishes/{id}/approve | 親が承認 |
| PUT | /api/wishes/{id}/unapprove | 承認取り消し |

### アルバム（/api/album）

| メソッド | パス | 概要 |
| --- | --- | --- |
| GET | /api/album | アルバム一覧取得（新しい順） |
| POST | /api/album | 叶えた記録を保存（写真パス・一言） |

### 設定（/api/settings）

| メソッド | パス | 概要 |
| --- | --- | --- |
| GET | /api/settings | 設定取得 |
| PUT | /api/settings | 設定更新 |
| POST | /api/settings/pin | PINコード設定・変更 |
| POST | /api/settings/pin/verify | PINコード検証 |

## 5. 画面設計（概要）

### 5.1 ホーム画面

* アプリ名・ロゴ表示
* 「🐉 冒険スタート」ボタン（→宝セット画面）
* 「⭐ 欲しいものリスト」ボタン
* 未承認のお願いごとがある場合バッジ表示
* 直近のアルバム記録をプレビュー表示

### 5.2 欲しいものリスト画面

* お願いごとカード一覧（欲しい度★表示・承認ステータス）
* 子どもモード：登録・削除・欲しい度変更
* 親モード：承認・承認取り消し
* 5個上限のプログレス表示（例：3/5）
* 登録フォーム（入力・★タップ）

### 5.3 宝セット画面（親用）

* OpenStreetMap地図表示（Leaflet.js）
* タップでボールを配置（ドラッグで移動）
* ボール数選択（3・5・7個）
* クエストあり / なし切り替え
* ボールごとのクエスト編集パネル（プリセット + 自由入力）
* 配置済みボール一覧サマリー
* 「セット完了」→確認画面→「冒険スタート！」

### 5.4 冒険画面（タブ構成）

**タブ①：レーダー**
* ドラゴンレーダー風円形UI
* 方向・距離をドットで表示（地図なし）
* ズームボタン（広域100m / 中域30m / 近接10m）
* 距離目盛り（設定スタイルに応じて表示）
* ボールカウンター（発見済み★ / 未発見数字）
* 発見時オーバーレイ演出

**タブ②：クエスト**
* 発見したボールのクエスト一覧
* 子どもチェック（👧）・親承認（👨）の2段階
* ステータス表示（未着手 / 子チェック済⏳ / 親承認済✅）
* 全承認でシェンロンタブが解放

**タブ③：シェンロン**
* 全条件達成までグレーアウト
* 達成後：シェンロン召喚演出
* 子ども：お願いごと選択（承認済みから最大3つ）
* 親：「かなえてあげる」承認
* 子ども：一言入力→アルバム記録
* アルバム確認画面

**タブ④：設定**
* レーダースタイル選択（⭕円 / ⊞方眼 / 🔀切替 / ✨重ね）
* ボール表示スタイル選択（4種）
* ボール数・クエスト設定
* 親モード認証設定（WebAuthn / PIN）

### 5.5 アルバム画面

* カード一覧（叶えたお願いごと・日付・写真・一言）
* タップで拡大表示
* 写真なしの場合はプレースホルダー表示

## 6. 認証設計（親モード切り替え）

```
子ども/親モード切り替えボタンタップ
　　↓
WebAuthn対応チェック
　　├── 対応：Face ID / Touch ID / 指紋認証
　　└── 非対応：PINコード（4桁）入力画面
　　　　　↓
　　　　認証成功 → 親モードへ切り替え
　　　　認証失敗 → エラー表示・子どもモードのまま
```

* PINはbcryptでハッシュ化してDB保存
* WebAuthnはブラウザ標準のWeb Authentication APIを利用
* 認証セッションは30分間有効（以降は再認証）

## 7. GPS・レーダー処理フロー

```
Geolocation API（watchPosition）
　　↓ 1秒以内に更新
現在位置（緯度・経度）取得
　　↓
各ボールとの距離・方向を計算
　　├── 距離計算：Haversine公式（メートル精度）
　　└── 方向計算：Math.atan2で角度を算出
　　↓
レーダーUI更新
　　├── ドット位置の更新（ズームに応じてスケーリング）
　　├── 点滅速度の更新（距離に応じて変化）
　　└── 音のテンポ更新（Web Audio API）
　　↓
発見判定
　　└── 距離 ≤ 5メートル → 発見演出
```

## 8. 技術スタック

| レイヤー | 技術 | 用途 |
| --- | --- | --- |
| フロントエンド | HTML / CSS / JavaScript | SPA構成 |
| 地図 | Leaflet.js + OpenStreetMap | 宝セット画面のみ |
| GPS | Geolocation API（watchPosition） | リアルタイム位置取得 |
| 音 | Web Audio API | 距離フィードバック |
| カメラ | MediaDevices API | アルバム写真撮影 |
| 認証 | Web Authentication API（WebAuthn） | 親モード切り替え |
| バックエンド | Python（FastAPI） | API・DB管理 |
| DB | SQLite（開発）→ PostgreSQL（本番） | 永続化データ |
| パスワード | bcrypt（passlib） | PINハッシュ化 |
| ホスティング | Railway | 本番環境 |
| 開発支援 | Claude Code | 実装 |
